# PixelTravelMap 简化体验与用户测试方案

日期：2026-07-22  
阶段：Discover  
决策：先完成最小、可逆的流程减法，再测试核心任务；不先做完整视觉重设计，也不直接拿当前复杂界面消耗正式用户样本。

## 1. 为什么不是“先改完”或“完全不改就测”

当前线上 Builder 首屏同时展示导入、定位、行程信息、地图预览、Day 编辑、15 列地点表格和多种下载选项。用户尚未得到结果，就需要理解坐标、交通类型、日期、预约、缓冲、poster 和 artifact 等多个概念。这是可由启发式检查直接发现的信息架构过载，无需真人再次证明。

但以下问题仍不能靠项目方自行决定：

- 用户最自然的输入是 Word、聊天文本还是一句简要描述。
- 自动识别错到什么程度会失去信任。
- 用户是否愿意确认城市或歧义地点。
- 用户首先需要总览地图、次日简报还是旅行后 poster。
- 像素视觉能否帮助理解或真实分享。

因此采用“最小减法原型 -> 真人任务测试 -> 再决定正式 UI”的顺序。

## 2. 本轮唯一核心任务

> 上传或粘贴一份已有行程，在尽量少的确认后得到一张可理解、可分享的旅行总览。

本轮不验证从零生成攻略、实时多人协作、精准导航、在线购票、美食推荐或复杂旅行记录。

## 3. 简化后的三步流程

### 第一步：给我行程

- 首屏只保留 Word 上传、一个大文本框和一个主按钮“整理我的行程”。
- 示例、隐私说明和支持格式放在次级入口。
- 不提前展示地图、空表格、下载按钮和技术设置。

### 第二步：确认少量问题

- 先按 Day 和城市生成草稿，再把真正阻塞生成的问题集中显示。
- 地图只要求每日城市或区域的大致方位；小景点默认归入城市，不要求每个 POI 填经纬度。
- 自动地点识别只在用户主动允许后运行；只要求确认歧义项，不逐点手填坐标。
- 交通、类别、停留、预约凭证和缓冲等字段移入“编辑更多细节”。
- 用按天卡片替代 15 列表格，一次只展开一天。

### 第三步：得到结果

- 默认先展示总行程地图和按天摘要。
- 主操作只保留“分享明日安排”和“保存旅行地图”。
- JSON、独立 HTML、总/每日/记录 poster 放入统一的“更多导出”菜单。
- 明确说明地图表示大致方位和旅行叙事，不承担精准导航。

## 4. 原型范围

设计先制作低成本可点击原型，不直接重写完整渲染器。原型至少覆盖：

1. 粘贴一份 3 天传统行程。
2. 系统按城市生成草稿。
3. 用户确认一个同名地点。
4. 生成总览。
5. 打开并分享第二天简报。

高级编辑、旅行记录和全部下载能力可以保留，但不出现在首次主流程。

## 5. 第一轮真人任务测试

参与者：5 名近期规划过 2 天以上自由行的人，其中至少 3 名有同行者。形成性测试不用于推断市场比例。

每人使用自己的可脱敏行程完成：

1. 导入已有行程并得到第一张可用地图。
2. 发现并修正一个地点或日期问题。
3. 找到第二天安排并说明如何发给同行者。

记录：

- 是否无需帮助完成。
- 到第一张可用地图的时间。
- 被要求确认和手填的字段数。
- 停顿、回退、误解和放弃点。
- 是否理解“故事地图不等于导航地图”。
- 分享时仍需回到微信、地图或原文寻找什么。

## 6. 本轮 Gate

以下是测试前的工作判据，不是已经取得的结果：

- 至少 4/5 参与者能在无指导下生成首张地图。
- 典型行程到首张地图不超过 3 分钟。
- 快乐路径不要求手填经纬度，非歧义地点无需逐项确认。
- 至少 4/5 能找到次日集合、交通和注意事项。
- 没有人把故事地图误认为精确导航。

未达到时继续简化输入和确认流程，不先增加更多 poster、推荐或 Agent 功能。

## 7. 设计与研发工作包

### 产品

- 输入：当前 Builder、合成研究审计、5 个真人任务。
- 输出：三步主流程、非目标、测试提纲、预注册 Gate。
- 验收：每个首次出现的字段都能说明为什么此刻必须填写。

### 设计

- 输入：三步流程和核心任务。
- 输出：桌面与手机低保真原型、空态、解析中、歧义确认、生成成功和错误状态。
- 验收：首屏只有一个主任务；高级功能逐步披露；不出现 15 列表格。

### 研发

- 输入：测试通过后的原型和字段优先级。
- 输出：城市级锚点、按天卡片、阻塞问题队列、更多导出菜单。
- 验收：保留现有离线 artifact 能力；旧 JSON schema 可兼容；高级字段不丢失。

### 研究

- 输入：可点击原型、真实行程、测试脚本。
- 输出：逐场记录、任务结果、失败路径、反例和下一 Gate 建议。
- 验收：合成 Agent 结果不得写成真人反馈；相反证据不得被删除。
